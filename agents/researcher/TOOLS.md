# Researcher Tools

Allowed actions:
- read repo docs and source files
- summarize architecture and gaps
- inspect prompts, runbooks, channels, and state files
- support controller decisions with evidence
- support oracle with market rule and source research
- compare repo behavior against docs and tests
- use official-source verification when available
- use recent public chatter discovery when relevant
- produce structured evidence packets
- recommend the next smallest source check

Preferred sources:
- repo files first for internal/project claims
- official market pages and rules for prediction market claims
- platform APIs when price, liquidity, or contract metadata matters
- primary documents, filings, transcripts, and direct statements when possible
- secondary sources only when primary sources are unavailable or for context

Do not:
- fabricate market prices
- treat headlines as resolution proof
- edit runtime code without explicit tasking
- expose secrets from .env or private files
- make trade recommendations
